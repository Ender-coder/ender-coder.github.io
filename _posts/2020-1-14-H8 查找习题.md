### 1、对于一个有n个元素的线性表，若采用顺序查找方法时的平均查找长度是什么？若结点是有序的，则采用折半查找法时的平均查找长度是什么？
&emsp;&emsp;顺序查找方法的平均查找长度为 $\frac{n+1}{2}$。  
&emsp;&emsp;折半查找方法的平均查找长度为 $\log_2(n+1)-1$。  


### 2、设查找表采用单链表存储，请分别写出对该表进行顺序查找的静态查找和动态查找的算法。
&emsp;&emsp;静态查找：
```cpp
SSTable* StaticSearch(SSTable *ST, KeyType key){
    SSTable *p = ST;
    while(p != NULL){
        if(p->data == key) return p;
        else p = p->next;
    }
    return NULL;
}
```
&emsp;&emsp;动态查找：
```cpp
SSTable* DymamicSearch(SSTable *ST, KeyType key){
    SSTable *p = ST;
    while(p != NULL){
        if(p->data == key) return p;
        else p = p->next;
    }
    SSTable *newST = new SSTable(key, NULL);
    ST.insert(newST);
    return NULL;
}
```


### 3、试比较哈希表构造时几种冲突处理方法的优点和缺点。
&emsp;&emsp;①开放地址法  
&emsp;&emsp;&emsp;&emsp;线性探测法：优点：只要散列表未满，总能找到一个不冲突的散列地址。  
&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;缺点：每个产生冲突的记录被散列到离冲突最近的空地址上，从而又增加了更多的冲突机会。  
&emsp;&emsp;&emsp;&emsp;二次探测法：优点：探测序列跳跃式地散列到整个表中，不易产生冲突的“聚集”现象。  
&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;&emsp;缺点：不能保证探测到散列表的所有地址。  
&emsp;&emsp;②再哈希法  
&emsp;&emsp;&emsp;&emsp;优点：不易产生冲突的“聚集”现象。  
&emsp;&emsp;&emsp;&emsp;缺点：计算时间增加。  
&emsp;&emsp;③链地址法  
&emsp;&emsp;&emsp;&emsp;优点：不产生冲突的“聚集”现象；删除记录也很简单。  
&emsp;&emsp;&emsp;&emsp;缺点：指针需要额外的空间。  
&emsp;&emsp;④建立公共溢出区  
&emsp;&emsp;&emsp;&emsp;优点：不产生冲突的“聚集”现象。  
&emsp;&emsp;&emsp;&emsp;缺点：需要建立额外的溢出表。
